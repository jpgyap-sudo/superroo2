## Scripts

### `scripts/sync-health.mjs`
Generates JSON and text health reports. Integrates with the sync daemon.

### `scripts/sync-monitor-agent.mjs`
Autonomously monitors and fixes sync issues:
- Retries failed VPS syncs with exponential backoff
- Resolves timestamp-based conflicts automatically
- Restarts crashed sync daemon
- Cleans up stale lock files

### `scripts/diagnose-vps-sync.mjs`
Tests each VPS endpoint independently for connectivity debugging.

## Conflict Resolution

When conflicts are detected, they are stored in `memory/.sync-conflicts.json` and can be:
1. Auto-resolved (timestamp-based) by the monitor agent
2. Manually reviewed via `scripts/sync-conflict-resolver.mjs --status`

## VPS Sync Queue

Failed VPS syncs are queued in `~/.superroo/sync-state/<project>-vps-queue.json`:
- Max 5 retry attempts
- Exponential backoff (2s base, 60s cap)
- Never blocks local consolidation

## Monitor Actions Log

All autonomous actions are logged to `memory/.monitor-actions.jsonl`:
- Action type
- Timestamp
- Reason for action
- Result (success/failure)

## Dashboard Integration

Add to your CI/CD or run manually:

```bash
# See current health
node scripts/sync-health.mjs

# See monitor actions
node scripts/sync-monitor-agent.mjs --status

# Run one health check cycle
node scripts/sync-monitor-agent.mjs --once
```
